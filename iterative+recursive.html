迭代算法与递归算法对比

概念

什么是迭代？

迭代算法是一种通过重复执行某个操作来解决问题的算法。它通过不断地重复某个过程，直到满足某个条件为止。

特性：重复执行某个操作。

什么是递归？

递归算法是一种在函数内部调用自身的算法。它将一个大问题分解为多个子问题，通过递归的方式解决。

特性：调用函数，将问题分解为多个子问题。

迭代与递归的优缺点

特性 迭代算法 递归算法
优点 内存使用较少 代码简洁，易于理解
计算过程可控，避免栈溢出 能够简洁地表示复杂问题的分解过程
缺点 代码可能较长，易出错 可能导致栈溢出，特别是递归深度过大时
某些问题难以通过简单的迭代实现 可能会增加程序的运行时间

迭代与递归的应用场景

迭代的应用

  • 动态规划问题

    如：Dijkstra 算法用于解决最短路径问题、最长公共子序列（LCS）则用来查找所有序列中最长子序列等。

  • 数学问题

    可解决阶乘、最大公约数、指数运算等。

  • 排序算法

    如：冒泡排序、选择排序、插入排序等。

递归的应用

  • 树的遍历

    如：前序遍历、中序遍历和后序遍历。

  • 回溯算法

    回溯算法是一种通过穷举所有可能的解，逐步构建解决方案，并在发现当前解不符合要求时“回退”的算法。

  • 分治算法

    分治算法是将一个规模为N的问题分解为K个规模较小的子问题，这些子问题相互独立且与原问题性质相同。


迭代与递归的区别

特性 迭代算法 递归算法
执行方式 通过循环实现 通过函数调用自身实现的
内存使用 通常使用较少的内存 可能会导致栈溢出，尤其是递归深度较大时
代码复杂度 迭代算法的代码通常比递归算法更长 递归算法的代码更加简洁

迭代与递归算法实现斐波那契数列代码对比

迭代算法实现


def fib_iterative(n):
    a, b = 0, 1
    for i in range(2, n + 1):
        a, b = b, a + b
    return b

递归算法实现


def fib_recursive(n):
    if n <= 1:
        return n
    return fib_recursive(n-1) + fib_recursive(n-2)
